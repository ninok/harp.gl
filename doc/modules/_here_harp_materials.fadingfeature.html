addRenderHelper

  • addRenderHelper(object: THREE.Object3D, fadeNear: number | undefined, fadeFar: number | undefined, forceMaterialToTransparent: boolean, updateUniforms: boolean, additionalCallback?: undefined | function): void
    • Defined in harp-materials/lib/MapMeshMaterials.ts:246

    As threejs is rendering the transparent objects last (internally), regardless of their renderOrder value, we set the transparent value to false in the [[onAfterRenderCall]]. In onBeforeRender, the function [[calculateDepthFromCameraDistance]] sets it to true if the fade distance value is less than 1.

    Parameters

    • object: THREE.Object3D

      [[THREE.Object3D]] to prepare for rendering.

    • fadeNear: number | undefined

      The fadeNear value to set in the material.

    • fadeFar: number | undefined

      The fadeFar value to set in the material.

    • forceMaterialToTransparent: boolean

      If true, the material will be forced to render with blending set to true. May be false if the material is known to be transparent anyway.

    • updateUniforms: boolean

      If true, the fading uniforms are set. Not rquired if material is handling the uniforms already, like in a [[THREE.ShaderMaterial]].

    • Optional additionalCallback: undefined | function

      If defined, this function will be called before the function will return.

    Returns void

    • Defined in harp-materials/lib/MapMeshMaterials.ts:184
